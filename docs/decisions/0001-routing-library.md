# ADR-0001: ルーティングライブラリの採用

- ステータス: 採用
- 日付: 2026-07-16

## 背景

商品カードから商品詳細ページ（`/product/:id`）へ遷移し、一覧へ戻る機能を実装する。
`/product/:id` という URL を持ち、ブラウザの戻る/進む・URL 直打ち・共有が
自然に機能する画面遷移が求められた。

## 決定

`react-router-dom`（v7 系）を採用し、`/` と `/product/:id` の URL ルーティングを実装する。

## 理由

- URL とアプリ状態が同期し、戻る/進む・直リンク・共有が追加実装なしで成立する。
- `BrowserRouter` / `Routes` / `Link` / `useParams` はデファクトで学習コストが低く、
  ワークショップ教材として読みやすい。
- Node 20+ / React 18 と互換（`engines: node>=20`、`peerDependencies: react>=18`）。

## 却下した代替案

- History API による自前ルーティング: 依存を増やさない利点はあるが、URL 解析・
  戻る/進む・フォールバック処理を自作する必要があり、教材の見通しが悪くなる。

## 影響

- ランタイム依存が 4 パッケージ増加（gzip 後のバンドルは約 48KB）。
- SPA のため、詳細 URL の直打ち時は Vite dev/preview の index.html フォールバックに依存する。
