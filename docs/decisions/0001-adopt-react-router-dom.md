# ADR-0001: 画面遷移に react-router-dom を採用する

- ステータス: 採用
- 日付: 2026-07-16

## 背景

商品カードから商品詳細ページ（`/product/:id`）へ遷移し、一覧へ戻る機能が必要になった。
これまでルーティングは無く、`App` の状態のみで単一画面を描画していた。

## 決定

ランタイム依存として `react-router-dom`（v7 系）を採用し、`/` と `/product/:id`
を URL ルーティングで扱う。

## 理由

- 詳細ページを実 URL（`/product/:id`）として表現でき、ブラウザの戻る操作や
  ブックマーク・共有と自然に整合する。
- React 18 / Node 20+ と互換（peer: react >=18, engines: node >=20）。
- デファクトのライブラリで学習コストが低く、ワークショップ教材に適する。

## 却下した代替案

- 自作ルーティング（History API + 状態管理）: 依存を増やさない利点はあるが、
  URL 同期・戻る操作・パラメータ解析を自前実装する保守コストが見合わない。

## 影響

- 依存が 4 パッケージ増加（`react-router-dom` と推移的依存）。型定義は同梱で
  `@types/*` は不要。
- ルート定義は `App` に集約。ブラウザ実行前提で Node 専用 API は導入しない。
